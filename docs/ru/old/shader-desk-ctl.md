# Руководство по использованию shader-desk-ctl

`shader-desk-ctl` — это утилита командной строки, предоставляющая интерфейс для динамического управления работающим графическим ядром. В отличие от простых конфигураторов, утилита выступает в роли клиента для удаленного выполнения Lua-кода. 

Через `shader-desk-ctl` вы можете на лету изменять состояние движка: от переопределения единичных параметров и вызова встроенных функций модуля `ctl`, до инъекции собственных сложных Lua-скриптов и функций для последующего использования.

## Архитектура взаимодействия

Взаимодействие с графическим ядром осуществляется через неблокирующий UNIX-сокет (поток `SOCK_STREAM`). Утилита автоматически обрабатывает сетевые нюансы:
* **Форматирование пакетов:** Все команды автоматически завершаются нуль-терминатором (`\0`), необходимым для парсера движка. При получении длинных ответов (например, JSON-объектов) поток считывается до нуль-терминатора, гарантируя целостность пакета.
* **Безопасность сеанса:** Установлен жесткий таймаут в 2 секунды на операции ввода-вывода. В случае зависания ядра (deadlock) утилита корректно завершит работу (вернув код `1`), не блокируя вызывающий процесс, например, оконный менеджер.
* **Обработка вывода и коды возврата:** При успешном выполнении возвращается код `0`, при ошибке — `1`. Стандартный ответ `OK` от ядра подавляется, запрошенные данные направляются в `stdout`, а ошибки выполнения Lua (маркер `LUA_ERR:`) выделяются красным цветом и перенаправляются в `stderr`.

## Способы передачи Lua-кода

Утилита поддерживает два варианта передачи инструкций, что делает ее универсальной как для быстрых горячих клавиш, так и для сложных сценариев автоматизации.

### 1. Через аргументы командной строки (Inline)
Этот способ оптимален для интеграции в конфигурационные файлы оконных менеджеров (Hyprland, Sway), статус-бары (Waybar, Eww) или для создания горячих клавиш. Код передается как строковый аргумент при вызове утилиты.

**Пример:** Вызов встроенного метода для переключения параметра на лету.
```bash
shader-desk-ctl "ctl.toggle('enable_stripes', 'eDP-1')"
```

### 2. Через стандартный ввод (stdin / Pipe)
Если утилита вызывается без аргументов, она переходит в режим чтения из конвейера. Это позволяет передавать многострочные скрипты, содержащие логические ветвления, циклы или объявления новых функций, а также использовать вывод других программ.

**Пример:** Передача файла со скриптом.
```bash
cat update_theme.lua | shader-desk-ctl
```

## Сценарии использования

Поскольку `shader-desk-ctl` выполняет любой валидный Lua-код в контексте работающего движка, возможности не ограничиваются жестко заданным списком команд.

### Использование встроенного модуля ctl
Графическое ядро предоставляет модуль `ctl`, содержащий готовые функции (хелперы) для типовых задач:
* **Установка параметров:** `ctl.set(param, value, output, tag)` — устанавливает конкретное значение. Использование `*` вместо имени дисплея применяет параметр ко всем мониторам.
* **Циклическое переключение:** `ctl.cycle(param, values_list, output, tag)` — удобно для смены цветовых схем по нажатию одной кнопки.
* **Уведомления (Вспышки):** `ctl.flash(color_rgb, duration_sec, output, tag)` — временное изменение цвета с плавным возвратом, реализованное через аппаратные таймеры `epoll`.
* **Остановка анимации:** `ctl.freeze(state, output, tag)` — замораживает анимацию, что полезно для экономии энергии.
* **Получение статуса:** `ctl.status()` — возвращает JSON-строку с информацией о мониторах и слоях.
* **Управление провайдерами:** `ctl.provider(name, enabled)` — включение/отключение фоновых демонов сбора данных.
* **Интеграция с темами:** `ctl.theme(mode, output)` для базовых светлых/темных тем и `ctl.pywal(output)` для динамического чтения палитры из `~/.cache/wal/colors.json`.

### Написание и инъекция собственных функций
Вы можете объявить собственную функцию на Lua, которая будет сохранена в памяти графического ядра, и вызывать ее в дальнейшем при наступлении системных событий.

**Пример скрипта (`custom_logic.lua`):**
```lua
-- Объявляем пользовательскую функцию в глобальной области видимости ядра
function apply_gaming_mode()
    ctl.freeze(true, '*')
    ctl.set('wireframe_mode', false, '*')
    print("Gaming mode activated")
end

-- Немедленный вызов при загрузке скрипта
apply_gaming_mode()
```
Загрузка логики в ядро: `cat custom_logic.lua | shader-desk-ctl`
Последующий вызов (например, по сочетанию клавиш): `shader-desk-ctl "apply_gaming_mode()"`

### Динамическая генерация кода
Использование стандартного ввода позволяет генерировать Lua-код на лету с помощью bash-скриптов на основе системных метрик. Например, можно читать уровень заряда батареи и динамически формировать Lua-команды для снижения интенсивности визуальных эффектов графического ядра.
