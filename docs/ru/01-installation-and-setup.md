# Установка и базовая настройка

В данном руководстве описан процесс компиляции Shader Desk из исходного кода, установки компонентов в систему и настройки автоматического запуска вместе с вашим Wayland-композитором.

## 1. Подготовка сборочной среды (Зависимости)

Для компиляции графического ядра и автономных демонов потребуются заголовочные файлы Wayland, EGL, а также ряд системных библиотек. Установите необходимые пакеты в зависимости от вашего дистрибутива.

**Arch Linux / Manjaro:**
```bash
sudo pacman -S base-devel cmake pkgconf wayland wayland-protocols luajit fftw libpulse libevdev
```

**Debian / Ubuntu:**
```bash
sudo apt update
sudo apt install build-essential cmake pkg-config libwayland-dev wayland-protocols libegl1-mesa-dev libgles2-mesa-dev libluajit-5.1-dev libfftw3-dev libpulse-dev libevdev-dev
```

**Fedora:**
```bash
sudo dnf install gcc-c++ cmake pkgconf wayland-devel wayland-protocols-devel mesa-libEGL-devel mesa-libGLES-devel luajit-devel fftw-devel pulseaudio-libs-devel libevdev-devel
```

## 2. Компиляция проекта

1. Склонируйте репозиторий и перейдите в директорию проекта:
```bash
git clone https://github.com/your-username/shader-desk.git
cd shader-desk
```

2. Сконфигурируйте проект с помощью CMake:
```bash
cmake -B build -DCMAKE_BUILD_TYPE=Release
```

3. Запустите процесс сборки (флаг `-j` использует все доступные ядра процессора):
```bash
cmake --build build -j$(nproc)
```

### Дополнительные флаги сборки (Опционально)
Если вам нужно только графическое ядро, и вы не планируете использовать интерактивные аудиовизуальные эффекты или реакцию на мышь, вы можете отключить сборку фоновых демонов для экономии ресурсов:
* `-DBUILD_AUDIO_DAEMON=OFF` — отключает сборку демона анализа спектра (PulseAudio/PipeWire).
* `-DBUILD_EVDEV_DAEMON=OFF` — отключает сборку демона захвата устройств ввода.
* `-DENABLE_PROFILING=ON` — включает интеграцию телеметрии Tracy (предназначено только для разработчиков).

## 3. Установка в систему

Рекомендуемый способ использования Shader Desk — системная установка. Проект спроектирован в строгом соответствии со стандартом иерархии файловой системы Linux (FHS).

Для установки выполните:
```bash
sudo cmake --install build
```

В результате файлы будут распределены следующим образом:
* `/usr/bin/interactive-wallpaper` — исполняемый файл графического микроядра.
* `/usr/bin/shader-desk-run` — bash-обертка для корректного старта демонов и ядра.
* `/usr/lib/shader-desk/plugins/` (или `lib64`) — скомпилированные `.so` библиотеки плагинов.
* `/usr/share/shader-desk/` — системные шейдеры, Lua-шаблоны и скрипты.

**Портативный режим:** Если вы являетесь разработчиком или просто хотите протестировать движок без системной установки, вы можете запускать скомпилированный бинарный файл напрямую из директории `./build/interactive-wallpaper`.

## 4. Системные права доступа (Критический этап)

Для обеспечения нулевой задержки (zero-latency) при обработке движений мыши, демон `evdev-pointer-daemon` считывает данные напрямую из устройств ввода ядра Linux (`/dev/input/event*`), минуя оконный менеджер.

По умолчанию обычный пользователь не имеет прав на чтение этих устройств. **Если не выполнить этот шаг, интерактивность обоев работать не будет.**

Добавьте вашего пользователя в системную группу `input`:
```bash
sudo usermod -aG input $USER
```

**Важно:** После выполнения этой команды необходимо полностью завершить сеанс (Log out) и войти в систему заново, либо перезагрузить компьютер, чтобы новые права вступили в силу.

## 5. Автозагрузка и интеграция с системой

Для автоматического старта обоев при входе в систему выберите один из двух вариантов.

### Вариант А: Интеграция через Systemd (Рекомендуется)
Проект поставляется с готовым файлом пользовательского сервиса `shader-desk.service`. 
Пользовательские сервисы Systemd часто стартуют до того, как оконный менеджер инициализирует графическую сессию и создаст переменную `WAYLAND_DISPLAY`. Чтобы сервис не завершился с ошибкой, оконный менеджер должен пробросить переменные среды.

1. Добавьте команду проброса переменных в конфигурацию вашего композитора.
*Для Hyprland (`~/.config/hyprland/hyprland.conf`):*
```text
exec-once = systemctl --user import-environment WAYLAND_DISPLAY XDG_CURRENT_DESKTOP
```
*Для Sway (`~/.config/sway/config`):*
```text
exec systemctl --user import-environment WAYLAND_DISPLAY XDG_CURRENT_DESKTOP
```

2. Включите и запустите сервис Systemd:
```bash
systemctl --user enable --now shader-desk.service
```

### Вариант Б: Запуск напрямую из оконного менеджера
Если вы не используете Systemd для управления пользовательскими сессиями, вы можете запускать Shader Desk напрямую через скрипт-обертку `shader-desk-run`. Эта обертка корректно запустит фоновые демоны сбора данных и графическое ядро, а при выходе безопасно завершит все дочерние процессы.

*Для Hyprland:*
```text
exec-once = shader-desk-run
```

*Для Sway:*
```text
exec shader-desk-run
```
