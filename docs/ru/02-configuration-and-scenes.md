# Конфигурация и Сцены (Lua API)

Shader Desk использует многоуровневую систему конфигурации на базе языка Lua. Вы можете использовать движок как простую утилиту для установки одного эффекта на фон, так и как сложный маршрутизатор, где каждый монитор живет своей жизнью и реагирует на системные события.

Вам не нужно знать C++ или GLSL для управления движком. Вся логика вынесена в Lua-скрипты.

## 1. Инициализация рабочего пространства

Движок поставляется с системными настройками по умолчанию (обычно в `/usr/share/shader-desk/`). Чтобы начать кастомизацию, создайте собственное изолированное рабочее пространство (Sandbox), которое не затрется при обновлении системы.

Выполните в терминале:

```bash
interactive-wallpaper --init-workspace
interactive-wallpaper --init-config
```

Эти команды создадут директорию `~/.config/interactive-wallpaper/` со следующей структурой:

```text
~/.config/interactive-wallpaper/
├── init.lua                 # Главный файл загрузки
├── scenes/                  # Директория с вашими визуальными сценами
├── plugins/                 # Глобальные настройки плагинов
└── effects/                 # Исходные коды плагинов для редактирования
```

## 2. Быстрый старт (Минимальная установка)

Если вам не нужна сложная логика, разные обои на разных мониторах и многослойность, вы можете использовать движок в минимальном режиме. Любой эффект самодостаточен и может работать в одиночку.

Откройте файл `~/.config/interactive-wallpaper/init.lua` и сотрите все лишнее. Идеальный минимальный конфиг выглядит так:

```lua
core = core or {}

-- 1. Просто указываем, какой плагин отображать на всех экранах
core.active_effect = "Hilbert Cube"

-- 2. (Опционально) Включаем провайдеры данных, если плагин их требует
core.providers = {
    ["Evdev Pointer Provider"] = { enabled = true }
}
```

При такой настройке движок автоматически прочитает параметры этого плагина из файла `plugins/hilbert_cube.lua` и применит их.

## 3. Продвинутая маршрутизация и Сцены

Для сложных сетапов (несколько слоев, разные эффекты на разных мониторах) используется система **Сцен**.

### 3.1. Маршрутизация экранов
В `init.lua` вместо `active_effect` используется таблица `core.outputs`:

```lua
core = core or {}

core.outputs = {
    -- Fallback: применяется ко всем мониторам, которые не указаны явно
    ["*"] = core.load_scene("liquid_desktop"),
    
    -- Явное указание (имена зависят от вашего композитора, например 'eDP-1')
    ["HDMI-A-1"] = core.load_scene("cyberpunk_city")
}
```

### 3.2. Анатомия Сцены (`scenes/*.lua`)
Сцена — это самодостаточный модуль. Она определяет список слоев (плагинов) и содержит локальные настройки.

Пример `scenes/cyberpunk_city.lua`:

```lua
return {
    layers = {
        {
            effect = "Universal Background", 
            tag = "bg",                      -- Уникальный тег для обращения из кода
            settings = {                     
                color_top = { 0.02, 0.03, 0.05 }
            }
        },
        {
            effect = "ShaderToy Sandbox",
            tag = "main_visual",
            settings = { 
                shader_file = "city_grid.glsl" 
            }
        }
    }
}
```
**Важно:** Обязательно задавайте `tag` каждому слою. Это семантическое имя (например, `bg` или `main_visual`), по которому ваши скрипты смогут находить нужный плагин в памяти.

## 4. Настройка плагинов и Выбор шейдеров

Один C++ плагин — это не всегда один шейдер. Плагин выступает в роли "драйвера", который управляет памятью, а отрисовывать он может разные GLSL-программы.

Когда вы запускаете `--init-config`, в папке `plugins/` генерируются файлы с заводскими настройками. 

### 4.1. Смена шейдера "на лету"
Многие плагины специально экспортируют параметр для переключения шейдерного файла или темы. 

**Пример 1: Плагин ShaderToy.** Он позволяет загружать любой совместимый GLSL файл из своей папки `shaders/`.
В `plugins/shadertoy_sandbox.lua`:
```lua
p.shader_file = "my_custom_shader.glsl" -- Имя файла в папке shaders/ бандла
```

**Пример 2: Плагин IcoSphere.** Имеет встроенные вариации.
В `plugins/icosahedron_sphere.lua`:
```lua
p.shader_theme = "harmonics" -- Загрузит harmonics_vert.glsl и harmonics_frag.glsl
```

> **Правило каскадирования:** Приоритет применения настроек выглядит так:
> `Значение из C++ по умолчанию` -> перекрывается -> `plugins/*.lua` -> перекрывается -> `scenes/*.lua (секция settings)` -> перекрывается -> `Динамический вызов через API`.

## 5. Хуки и Анимация (Таймеры)

Shader Desk предоставляет два механизма для запуска пользовательской логики: покадровый хук и асинхронные таймеры.

### 5.1. Покадровый хук (`on_frame`)
Выполняется синхронно с частотой обновления вашего монитора (например, 144 раза в секунду). Идеально подходит для плавной математической анимации.

**Внутри Сцены:**
```lua
return {
    layers = { { effect = "Hilbert Cube", tag = "cube" } },
    state = { time = 0.0 }, -- Изолированная память сцены
    
    on_frame = function(self, dt, output_name)
        self.state.time = self.state.time + dt
        
        -- Вычисление пульсации цвета
        local r = 0.5 + 0.5 * math.sin(self.state.time)
        
        -- Обращаемся к слою по тегу и безопасно меняем параметры
        core.get_layer(output_name, "cube"):set("curve_color", {r, 0.0, 0.5})
    end
}
```

### 5.2. Асинхронные таймеры (`set_interval`)
Если вам нужно выполнять действие редко (например, менять обои каждые 5 минут или сделать вспышку на 300 миллисекунд), использовать `on_frame` расточительно. 

Для этого есть `core.set_interval(ms, callback)`. Под капотом он создает аппаратный `timerfd` и регистрирует его в ядре через `epoll`. Это гарантирует нулевую нагрузку на процессор в режиме ожидания.

**Внутри `init.lua` или `ctl.lua`:**
```lua
-- Выполнять функцию каждые 10 секунд (10000 мс)
local timer_id = core.set_interval(10000, function()
    -- Получаем доступ к слою на мониторе eDP-1
    local layer = core.get_layer("eDP-1", "bg")
    
    -- Генерируем случайный цвет
    local new_color = { math.random(), math.random(), math.random() }
    layer:set("color_top", new_color)
    
    -- Если функция возвращает nil, таймер продолжает работать.
    -- Если вернуть что-либо, таймер не остановится, но для явной остановки используйте:
    -- core.clear_interval(timer_id)
end)
```
*Примечание: Существует жесткий лимит в 32 одновременно активных таймера для защиты от утечек памяти.*

## 6. Продвинутые возможности

### 6.1. Пост-обработка (Post-Processing)
Плагины можно использовать как фильтры. Добавьте слой с флагом `is_postprocess = true`. Движок автоматически перенаправит рендер предыдущих слоев на вход этого плагина (создав Ping-Pong FBO).

```lua
layers = {
    { effect = "Hilbert Cube", tag = "cube" },
    { effect = "CRT ShaderToy", tag = "crt_filter", is_postprocess = true }
}
```

### 6.2. Изолированные профили запуска
Для использования разных конфигураций (например, тяжелые эффекты для работы от сети и статичный фон для батареи), создайте копию директории:

```bash
cp -r ~/.config/interactive-wallpaper ~/.config/interactive-wallpaper-battery
```
Отредактируйте настройки, а затем запускайте движок с флагом `--config`:
```bash
interactive-wallpaper --config ~/.config/interactive-wallpaper-battery
```

## 7. За пределами Lua: Мощь модульной архитектуры

Lua-конфигурация — это лишь "панель управления" движком. Она идеально справляется с маршрутизацией мониторов и плавной интерполяцией состояний, но истинная мощь Shader Desk кроется в его микроядерной архитектуре.

Движок строго разделяет логику управления (Lua), графический конвейер (C++/GLSL) и сбор системных данных (Standalone-демоны). Если вы ограничиваетесь только редактированием Lua-скриптов, вы упускаете возможность создавать по-настоящему глубокие и производительные интерактивные системы. 

Интерпретируемый язык не предназначен для симуляции физики 100 000 частиц, расчета гидродинамики (Navier-Stokes) или обработки массивов аудио-частот со скоростью 144 кадра в секунду. Архитектура Shader Desk позволяет вам спуститься на уровень нативного кода ровно в тех местах, где это необходимо, полностью сохраняя безопасность и стабильность основного событийного цикла Wayland.

### Экосистема и дистрибуция

Благодаря строгой C-ABI изоляции, каждый визуальный плагин или дата-провайдер не является просто куском кода, привязанным к конкретному конфигу пользователя. Это самодостаточный, портативный бандл (содержащий скомпилированный `.so` файл, папку `shaders/` и готовые `presets/`).

Такой дизайн делает компоненты максимально удобными для шеринга. Чтобы установить чужой эффект, пользователю достаточно просто скопировать его папку в свою директорию `~/.config/interactive-wallpaper/effects/`. 

Каждый написанный вами качественный плагин, портированный шейдер или полезный демон сбора метрик — это потенциально популярный самостоятельный проект. Мы призываем разработчиков делиться своими работами: лучшие и самые оптимизированные плагины агрегируются сообществом в официальном репозитории **shader-desk-awesome**, формируя единую экосистему интерактивных рабочих столов для Wayland.
