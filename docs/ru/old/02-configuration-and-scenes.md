# Конфигурация и сцены (Lua API)

Shader Desk использует многоуровневую систему конфигурации на базе языка Lua. Вы можете использовать движок как простую утилиту для установки одного эффекта на фон, так и как сложный маршрутизатор, где каждый монитор имеет свою сцену с несколькими слоями и фильтрами.

В этом разделе описана структура конфигурационных файлов, принципы каскадирования настроек, использование тегов и работа со сценами.

---

## 1. Структура конфигурации

Конфигурация Shader Desk состоит из набора Lua‑файлов, расположенных в пользовательской директории `~/.config/interactive-wallpaper/` (или в кастомной, указанной через `--config`).

```
~/.config/interactive-wallpaper/
├── init.lua                 # Главный файл загрузки
├── ctl.lua                  # Вспомогательный модуль для удалённого управления
├── providers.lua            # Конфигурация провайдеров данных
├── scenes/                  # Директория с визуальными сценами
│   ├── default.lua
│   ├── liquid.lua
│   └── ...
└── plugins/                 # Глобальные настройки плагинов
    ├── hilbert_cube.lua
    ├── ico_sphere.lua
    └── ...
```

После первого запуска или после вызова `interactive-wallpaper --init-workspace` и `--init-config` эти файлы создаются автоматически из встроенной виртуальной файловой системы (VFS).

---

## 2. Иерархия приоритетов настроек

Параметры плагина могут быть определены на нескольких уровнях. Движок применяет их в следующем порядке (от низшего к высшему):

1. **C++ значение по умолчанию** — жёстко задано в коде плагина.
2. **Глобальная конфигурация плагина** — файл `plugins/<plugin_name>.lua`.
3. **Настройки слоя в сцене** — секция `settings` в `scenes/<scene_name>.lua`.
4. **Динамическое изменение через API** — вызов `layer:set()` из Lua или через `shader-desk-ctl`.

Это позволяет, например, задать базовые цвета в глобальном конфиге, переопределить их для конкретного монитора в сцене, а затем динамически менять их из `on_frame` в зависимости от времени.

---

## 3. Теги и их роль

Каждый слой или фильтр в сцене может иметь **тег** (параметр `tag`). Тег выступает в роли первичного ключа для идентификации инстанса плагина на конкретном мониторе.

### Зачем нужны теги?

- **Обращение из Lua** — через `core.get_layer(output_name, tag)` вы получаете доступ к конкретному слою.
- **Горячая перезагрузка** — при изменении конфигурации ядро сопоставляет слои по тегу и переиспользует их, сохраняя состояние (например, накопленное время анимации).
- **Различение одинаковых плагинов** — если в сцене используется два экземпляра одного плагина (например, два куба с разными цветами), они **обязаны** иметь разные теги. Без уникальных тегов ядро не сможет их различить, и оба слоя будут ссылаться на один инстанс.

### Правила выбора тега

- Если тег не указан, он автоматически принимает значение имени эффекта (`effect`).
- Для уникальных экземпляров одного эффекта всегда задавайте явные теги.
- Тег может быть любой строкой, но рекомендуется использовать латиницу без пробелов.

---

## 4. Сцены (Scenes)

Сцена — это самодостаточный Lua‑модуль, который возвращает таблицу конфигурации. Он определяет список слоёв (визуальных эффектов) и их параметры.

### 4.1. Базовая структура сцены

```lua
local M = {
    fps_limit = 0.0,        -- ограничение частоты кадров (0 = без ограничения)
    fbo_scale = 1.0,        -- масштаб рендеринга (0.5 = половинное разрешение)
    
    layers = {
        {
            effect = "Solid Bg",          -- имя плагина
            tag = "background",           -- уникальный тег
            clear_depth = true,           -- очищать Z-буфер перед слоем
            settings = {                  -- параметры плагина
                gradient_type = 2,
                color_1 = {0.2, 0.3, 0.5}
            }
        },
        {
            effect = "Hilbert Cube",
            tag = "cube",
            settings = {
                curve_color = {1.0, 0.0, 0.0}
            },
            filters = {                   -- вложенные пост-эффекты
                {
                    effect = "Postprocess Effect",
                    tag = "glitch",
                    settings = {
                        intensity = 0.3,
                        variant = 1
                    }
                }
            }
        }
    },
    
    -- изолированное состояние сцены (доступно в on_frame)
    state = { time = 0.0 },
    
    -- покадровый хук
    on_frame = function(self, dt, output_name)
        self.state.time = self.state.time + dt
        -- обновление параметров через API
        core.get_layer(output_name, "cube"):set("rotation_speed", math.sin(self.state.time))
    end
}

return M
```

### 4.2. Параметры слоя

| Поле | Тип | Описание |
|------|-----|----------|
| `effect` | строка | Имя плагина (обязательно) |
| `tag` | строка | Уникальный идентификатор слоя (опционально) |
| `clear_depth` | boolean | Очищать Z-буфер перед слоем (по умолчанию `true` для первого слоя, далее `false` для 3D-корректности) |
| `settings` | таблица | Параметры плагина (соответствуют его `get_parameters()`) |
| `filters` | массив | Список вложенных пост-эффектов (структура аналогична слоям) |
| `preset` | строка | Имя пресета для автоматического применения (из папки `presets/` плагина) |

### 4.3. Вложенные фильтры (Nested Filters)

Фильтры — это слои, которые применяются **к результату рендеринга родительского слоя**, а не к общему буферу сцены. Это позволяет, например, наложить глитч-эффект только на 3D-куб, не затрагивая фон.

**Архитектурные особенности:**

- Фильтры наследуют Z-буфер родительского слоя через механизм Depth Blitting.
- Они используют тот же Ping‑Pong FBO, что и родительский слой, но не создают отдельный проход для всей сцены.
- Фильтры могут быть вложенными (фильтр поверх фильтра), создавая цепочки пост-обработки.

**Пример вложенности:**

```lua
layers = {
    {
        effect = "Icosahedron Sphere",
        tag = "planet",
        filters = {
            {
                effect = "Postprocess Effect",
                tag = "kaleidoscope",
                settings = { scale = 6.0 }
            },
            {
                effect = "Postprocess Effect",
                tag = "crt",
                settings = { variant = 1 }
            }
        }
    }
}
```

### 4.4. Покадровый хук `on_frame`

Функция `on_frame` выполняется синхронно с частотой кадров для каждого монитора. Она получает:

- `self` — таблицу сцены (позволяет хранить состояние).
- `dt` — дельта времени в секундах.
- `output_name` — имя монитора.

Внутри `on_frame` можно использовать `core.get_layer()` для обновления параметров в реальном времени.

## 5. Пресеты (Presets)

Пресеты — это готовые наборы параметров для плагина, хранящиеся в папке `presets/` его бандла. Они позволяют быстро переключаться между разными визуальными стилями.

### 5.1. Структура пресета

Файл пресета — это Lua‑таблица с теми же ключами, что и параметры плагина:

```lua
-- presets/cyberpunk.lua
return {
    wireframe_mode = true,
    background_color = {0.05, 0.01, 0.08},
    wireframe_color = {1.0, 0.0, 0.8},
    rotation_speed = 0.4,
    rotation_decay = 0.98
}
```

### 5.2. Применение пресета

В сцене:

```lua
{
    effect = "Icosahedron Sphere",
    preset = "cyberpunk"   -- автоматически загружает presets/cyberpunk.lua
}
```

Из командной строки:

```bash
shader-desk-ctl "ctl.preset('Icosahedron Sphere', 'cyberpunk', '*')"
```

### 5.3. Особенности слияния

При применении пресета его значения **не перезаписывают** уже существующие ключи в `settings`. Это позволяет задать в сцене свои значения поверх пресета, сохраняя остальные параметры.

---

## 6. Провайдеры данных

Конфигурация провайдеров данных (демонов, поставляющих системные метрики) вынесена в отдельный файл `providers.lua` и подключается в `init.lua`.

```lua
-- providers.lua
core.providers = {
    ["Evdev Pointer Provider"] = {
        enabled = true,
        mouse_sensitivity = 1.0,
        invert_x = false
    },
    ["Native FFTW Audio Provider"] = {
        enabled = true,
        smoothing = 0.85,
        volume_multiplier = 1.0
    }
}
```

Провайдеры можно включать/отключать на лету через `ctl.provider()`.

## 7. Примеры конфигураций

### 7.1. Минимальная конфигурация

```lua
-- init.lua
core = core or {}
core.active_effect = "Hilbert Cube"
core.providers = {
    ["Evdev Pointer Provider"] = { enabled = true }
}
```

### 7.2. Два монитора с разными сценами

```lua
core.outputs = {
    ["eDP-1"] = core.load_scene("liquid"),
    ["HDMI-A-1"] = core.load_scene("cyberpunk_city"),
    ["*"] = core.load_scene("default")
}
```

### 7.3. Сцена с несколькими слоями и фильтрами

```lua
-- scenes/example.lua
return {
    layers = {
        {
            effect = "Solid Bg",
            tag = "bg",
            settings = { gradient_type = 2, color_1 = {0.1, 0.2, 0.3} }
        },
        {
            effect = "Hilbert Cube",
            tag = "cube",
            clear_depth = true,
            settings = { hilbert_order = 4, curve_color = {0.0, 1.0, 0.0} },
            filters = {
                {
                    effect = "Postprocess Effect",
                    tag = "glitch",
                    settings = { intensity = 0.2, variant = 1 }
                }
            }
        }
    },
    state = { angle = 0.0 },
    on_frame = function(self, dt, output)
        self.angle = self.angle + dt * 0.5
        core.get_layer(output, "cube"):set("rotation_speed", math.sin(self.angle) * 0.5)
    end
}
```
